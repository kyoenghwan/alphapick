# 🤖 봇 정의 및 데이터 공급 체계 명세서 (Bot Specifications)

본 문서는 AlphaPick 시스템의 핵심인 AI 봇들의 분석 기법, 데이터 관측 범위, 그리고 리스크 제어 규칙을 정의합니다. 모든 봇은 정의된 3단계 데이터 패키지를 기반으로 독립적인 분석을 수행합니다.

---

## 0. 게임 규칙 및 75% 필승 배팅 전략 (Game Rules & Strategy)

본 시스템은 단순한 홀짝(50%) 게임이 아니라, **3개 요소(좌/우, 3/4줄, 홀/짝) 중 2개가 일치하면 승리**하는 특수 룰을 기반으로 합니다.

### 1. 실제 결과 (Actual Results) - 4가지 불변
게임에서 실제 발생 가능한 결과값입니다.
*   **L3X** (좌3짝)
*   **L4O** (좌4홀)
*   **R3O** (우3홀)
*   **R4X** (우4짝)

### 2. 예측 픽 (Prediction Picks) - 4가지 전략 픽
봇은 실제 결과를 맞추는 것이 아니라, 확률적으로 승리하는 아래 4개 픽 중 하나를 선택합니다.
*   **L3O** (좌3홀)
*   **L4X** (좌4짝)
*   **R3X** (우3짝)
*   **R4O** (우4홀)

### 3. 승리 조건 (2/3 매칭)
예측 픽의 3개 요소 중 **2개 이상**이 실제 결과와 일치하면 적중(Win)입니다. 
따라서 4개의 예측 픽 중 3개는 항상 승리하고, 1개만 패배하는 구조(75% 승률)를 가집니다.

## 1. 공통 데이터 패키지 구조 (3rd Layer Hierarchy)

모든 봇(01~20)은 분석 시 아래 3가지 계층의 데이터를 세트로 전달받아 입체적인 분석을 수행합니다.

| 계층 | 명칭 | 데이터 성격 | 제공 내용 |
| :--- | :--- | :--- | :--- |
| **계층 1** | **거시 족보 (Macro)** | 30일 누적 통계 | 요일/시간대별 기본 확률, 패턴 한계값(Max Length) 등 |
| **계층 2** | **롤링 요약 (Rolling)** | 최근 120~1440회차 | 단/중/장기 분석 그룹별 맞춤 요약 (Z-Score, 추세, 확률 수렴도 등) |
| **계층 3** | **실시간 생데이터 (Raw)** | 최근 15~40회차 | 각 봇의 기법에 최적화된 가변 윈도우 결과값 시퀀스 |

## 3. 계층별 상세 추출 및 요약 로직

### 계층 1: 거시 족보 (30일 배치 분석)
*   **요일별 확률**: 최근 30일 중 '오늘과 같은 요일'의 L/R 비중 추출.
*   **시간대별 가점**: 24시간을 1시간 단위로 구분하여 특정 시간대의 쏠림 가중치($\omega$) 산출.
*   **패턴 한계값**: 30일간 발생한 최장 '줄'과 '퐁당' 길이를 기록하여 예측 임계점으로 활용.

### 계층 2: 롤링 요약 (3~10분 주기 계산)
*   **단기용 (120~240회)**: 줄타기 전성비, 퐁당 평균 지속 마디, 상위 3개 출현 모양 추출.
*   **중기용 (480회 / 24시간)**: 구간별 승률 추이, 변동성 지수, 24시간 데일리 평균값 제공.
*   **장기용 (720~1440회 / 36~72시간)**: 
    *   **승률 수렴도(Convergence)**: 3일간의 데이터를 30일 거시 족보와 대조하여 현재의 확률적 위치 산출.
    *   **거대 파동 주기**: 3일 데이터를 4시간 단위 블록으로 구분하여 큰 흐름의 에너지 방향 수치화.
    *   **신뢰 구간 정의**: 1,000개 이상의 표본을 통해 소음이 제거된 '진짜 통계적 뼈대' 제공.

## 4. 최종 마스터 결정 및 패스(PASS) 로직

**BOT_20**은 단순히 L/R을 맞추는 것이 아니라, **"틀릴 확률이 높으면 돈을 지킨다"**는 원칙에 따라 아래 조건에서 **PASS**를 결정합니다.

*   **의견 불일치 (Divergence)**: 단기, 중기, 장기 그룹이 서로 다른 방향을 가리킬 때.
*   **저신뢰도 (Low Confidence)**: 16~18번 검증 봇들이 산출한 신뢰도 점수가 기준치 미달일 때.
*   **변동성 폭주 (Volatility)**: 최근 결과가 30일 족보 및 롤링 윈도우의 통계적 범위를 완전히 벗어났을 때.
*   **연패 방어 (Streak Defense)**: 특정 그룹의 최근 적중률이 급락하여 데이터 해석의 신뢰성이 깨졌을 때.

## 5. 리스크 제어 및 온도(Temperature) 규칙

봇의 예측 모델 신뢰도에 따라 생성 인자의 '창의성'과 '수학적 고정성'을 조절합니다.

*   **기본 상태 (Normal)**: 봇의 성격에 따라 $T = 0.3 \sim 0.7$ 유지.
*   **리스크 감지 (Risk/Loss)**: 연패 발생 시 해당 봇의 **온도를 0.1로 강제 하락**.
*   **효과**: 봇이 주관적 추론을 배제하고 수학적 확률이 가장 높은 보수적 데이터에만 집중하게 함.

## 7. 봇별 프롬프트 최적화 양식 (Prompt Templates)

모든 봇은 AI API 비용 절감과 응답 속도 향상을 위해 **최소한의 토큰**으로 **정해진 JSON 포맷**만 리턴하도록 설계됩니다.

### 7.1 공통 시스템 프롬프트 (System Instruction)
```text
You are AlphaPick's [BOT_ID], specialized in [Algorithm_Name].
Your goal is to analyze the provided 3-layer data and predict the optimal betting pick (L3O, L4X, R3X, R4O).
You MUST output strictly in JSON format.
Risk Control: If confidence is below 60%, output "PASS".
Temperature: [Current_Temp] (Lower this if on losing streak).
```

### 7.2 데이터 주입 양식 (User Prompt Example: ST Group)
**단기 봇에게는 '모양'을 보여주는 것이 핵심이므로 Raw Sequence를 그대로 줍니다.**
```text
[Layer 1: Macro]
- Today's L Probability: 51.2%
- Max Streak: 7

[Layer 2: Rolling Summary (Last 120)]
- Current Stripe Ratio: 40% (High)
- Pongdang Stability: Low

[Layer 3: Raw Data (Last 30)]
L,L,R,L,L,L,R,R,R,L,R,L, (Current)
```

### 7.3 데이터 주입 양식 (User Prompt Example: MT/LT Group)
**장기 봇에게는 1000개의 Raw Data 대신 '압축된 통계'를 주어 토큰을 절약합니다.**
```text
[Layer 1: Macro]
- 30-Day Avg Win-Rate: L(50.1%)

[Layer 2: Rolling Summary (Last 1440)]
- 3-Day Win-Rate: L(48.5%) -> Variance: -1.6% (Undervalued)

[Layer 3: Raw Data (Last 24h Compressed)]
- L Count: 230
- R Count: 250
- Current Segment (Last 50): L skewed (60%)
```

### 7.4 출력 JSON 포맷 (Output Schema)
모든 봇은 이 형식을 엄격히 따릅니다.
```json
{
  "bot_id": "BOT_04",
  "prediction": "L3O", // "L3O", "L4X", "R3X", "R4O", or "PASS"
  "confidence": 85,    // 0 ~ 100 Integer
  "risk_level": "LOW", // LOW, MEDIUM, HIGH
  "reason": "Z-Score +2.8 indicates extreme deviation. Regression to mean expected."
}
```
